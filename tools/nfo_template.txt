   [ system ]    Stock Commodore 64 + 1541-compatible drive, PAL.
                 Tested against 1541-Ultimate (the X2026 compo hardware).
                 No BASIC stub. Spindle 3.1 boots us straight in.

   [ sid    ]    8580 preferred.
                 The 8580 is the more REPRODUCIBLE chip across
                 hardware (digital filter, no per-unit cutoff drift),
                 and its cleaner filter character is what lets the
                 sidechain + LP wah + dark-phaser tricks in
                 interlude / greets / end land the same way
                 on every C64. The 6581 will still play but the
                 filter behaviour is unit-dependent.

   [ runtime ]   One-pass ~3:00 (~180 s boot to credits-roll-loops).
                 End credits then loop forever; the demo is over.

   [ ------- honest note on AI authorship ------- ]


   This release was written in three weeks by a human
   (Anus/deFEEST) collaborating with three large-language-model
   AI agents in a tight feedback loop:

     * Kloot   = Anthropic Claude (Opus 4.7), via Claude Code
     * Augurk  = OpenCode, the agent-coding tool
     * TL-Buis = OpenAI ChatGPT

   The C64 source is genuine 6502 in KickAssembler syntax. The
   AIs wrote most of the actual KickAssembler code — IRQ
   chains, FLD bounce, DYCP scroller, SID drum kit, parallax
   starfield, Stage F ping-pong zoom on the Kloot stars, the
   end-credits clean<->dark LFO modulation. The humans drove the
   process — picking effects, listening, reading what landed
   on screen, correcting bugs ("nope this is broken because
   <X>"), making aesthetic calls.

   The development loop was: edit -> ./build.sh -> autostart in
   a custom VICE-MCP build -> AI inspects the running emulator
   via JSON-RPC (screenshots, register reads, memory dumps,
   raster-line checkpoints) -> AI proposes a fix or a new
   effect -> commit -> repeat. ~180 commits over three weeks.

   We think this is novel in 2026 and worth being open about.
   The repo is public; every decision is in the commit log.

   The C64 itself doesn't know or care. The bytes are the
   bytes; PAL is PAL; the SID still hums. We hope it lands.
